# Quickstart

This guide will get you up and running with PicoSearch in just a few minutes. You'll learn how to perform basic fuzzy searches on your data.

## Installation

First, install PicoSearch using your preferred package manager:

<CodeGroup>

```bash npm
npm install @scmmishra/pico-search
```

```bash pnpm
pnpm add @scmmishra/pico-search
```

```bash yarn
yarn add @scmmishra/pico-search
```

</CodeGroup>

## Basic usage

<Steps>
  <Step title="Import the library">
    Import the `picoSearch` function from the package:

    ```typescript
    import { picoSearch } from "@scmmishra/pico-search";
    ```
  </Step>

  <Step title="Prepare your data">
    PicoSearch works with arrays of objects. Define your data structure:

    ```typescript
    interface Person {
      name: string;
      age: number;
    }

    const people: Person[] = [
      { name: "Alice", age: 25 },
      { name: "Bob", age: 30 },
      { name: "Charlie", age: 35 },
      { name: "David", age: 40 },
    ];
    ```
  </Step>

  <Step title="Perform a search">
    Call `picoSearch` with your data, search term, and the keys to search:

    ```typescript
    const searchTerm = "ali";
    const keys = ["name"];

    const results = picoSearch(people, searchTerm, keys);
    console.log(results);
    // Output: [{ name: "Alice", age: 25 }]
    ```
  </Step>
</Steps>

## Complete example

Here's a complete working example you can copy and run:

```typescript
import { picoSearch } from "@scmmishra/pico-search";

interface Person {
  name: string;
  age: number;
  city: string;
}

const people: Person[] = [
  { name: "Alice Johnson", age: 25, city: "New York" },
  { name: "Bob Smith", age: 30, city: "San Francisco" },
  { name: "Charlie Brown", age: 35, city: "Chicago" },
  { name: "David Lee", age: 40, city: "Boston" },
];

// Simple search
const results1 = picoSearch(people, "ali", ["name"]);
console.log("Search for 'ali':", results1);
// Output: [{ name: "Alice Johnson", age: 25, city: "New York" }]

// Subsequence matching
const results2 = picoSearch(people, "cg", ["city"]);
console.log("Search for 'cg':", results2);
// Output: [{ name: "Charlie Brown", age: 35, city: "Chicago" }]

// Multi-field search
const results3 = picoSearch(people, "bob", ["name", "city"]);
console.log("Search for 'bob':", results3);
// Output: [{ name: "Bob Smith", age: 30, city: "San Francisco" }]
```

## Understanding the results

PicoSearch returns an array of objects that match your search term, sorted by relevance:

- **Case insensitive:** Searches are case-insensitive by default
- **Subsequence matching:** You can type scattered characters (e.g., "ajn" matches "Alice Johnson")
- **Ranked by relevance:** Results are sorted with the best matches first
- **Original objects:** The returned objects are the same objects from your input array (not copies)

## Common patterns

### Searching multiple fields

Search across multiple properties of your objects:

```typescript
const results = picoSearch(people, "york", ["name", "city"]);
// Finds matches in both name and city fields
```

### Using weighted keys

Prioritize certain fields over others:

```typescript
const results = picoSearch(people, "john", [
  { name: "name", weight: 3 },  // Name is 3x more important
  { name: "city", weight: 1 },  // City has default weight
]);
```

### Adjusting the threshold

Control how strict the matching should be:

```typescript
// Stricter matching (fewer results)
const strictResults = picoSearch(people, "ali", ["name"], {
  threshold: 0.7,
});

// Looser matching (more results)
const looseResults = picoSearch(people, "ali", ["name"], {
  threshold: 0.2,
});
```

<Note>
The default threshold is 0.3, which provides a good balance between precision and recall for most use cases.
</Note>
