title: Weighted keys
description: Learn how to prioritize certain fields in your search results

By default, all keys in PicoSearch are weighted equally. Weighted keys allow you to make some fields more important than others when calculating match scores.

## Basic weighted keys

To specify a weight for a key, pass an object with `name` and `weight` properties instead of a string:

```typescript
const keys = [
  { name: "name", weight: 2 },
  "age" // default weight of 1
];
```

## How weights work

Weights are **relative** to each other. A key with weight 2 is considered twice as important as a key with weight 1:

```typescript
const objects = [
  { name: "John Doe", age: 25, city: "New York" },
  { name: "Jane Smith", age: 35, city: "San Francisco" },
  { name: "Bob Johnson", age: 45, city: "Los Angeles" },
];

const keys = [
  { name: "name", weight: 5 },
  { name: "city", weight: 1 },
];

const result = picoSearch(objects, "John", keys);
// John Doe ranks highest because "John" matches the name field,
// which has 5x more weight than city
```

<Note>
The actual numeric values don't matter, only their ratio. `{weight: 10}` and `{weight: 5}` have the same effect as `{weight: 2}` and `{weight: 1}`.
</Note>

## Practical example: Name vs city

Here's a real-world example where you want name matches to be more important than location matches:

<CodeGroup>

```typescript Heavy name weighting
const keys = [
  { name: "name", weight: 5 },
  { name: "city", weight: 1 },
];

const result = picoSearch(objects, "John", keys);
// "John Doe" ranks first (exact name match)
// "Bob Johnson" ranks second (partial name match)
```

```typescript Equal weighting
const keys = ["name", "city"];
// Both keys have weight 1 by default

const result = picoSearch(objects, "John", keys);
// Results may rank differently without emphasis on name
```

</CodeGroup>

## Mixing weighted and unweighted keys

You can mix string keys (default weight 1) with weighted keys in the same array:

```typescript
const keys = [
  { name: "title", weight: 3 },
  { name: "description", weight: 2 },
  "tags",  // weight: 1
  "category"  // weight: 1
];
```

This configuration means:
- Title matches are 3x more important than tags or category
- Description matches are 2x more important than tags or category
- Tags and category have equal importance

## Multi-word queries with weights

Weighted keys work seamlessly with multi-word queries:

```typescript
const data = [
  { name: "John Doe", city: "New York" },
  { name: "Jane Smith", city: "Boston" },
];

const results = picoSearch(data, "John New", [
  { name: "name", weight: 3 },
  { name: "city", weight: 2 },
]);
// [{ name: "John Doe", city: "New York" }]
```

<Tip>
For multi-word queries, each word finds its best score across all keys. Weights influence which field's match contributes more to the overall score.
</Tip>

## Common weighting strategies

Here are some typical weighting patterns:

<Tabs>
  <Tab title="User search">
    ```typescript
    const keys = [
      { name: "name", weight: 5 },
      { name: "email", weight: 3 },
      { name: "company", weight: 1 },
    ];
    ```
    Prioritizes exact name matches, then email, then company affiliation.
  </Tab>
  
  <Tab title="Product search">
    ```typescript
    const keys = [
      { name: "title", weight: 4 },
      { name: "sku", weight: 4 },
      { name: "description", weight: 2 },
      { name: "category", weight: 1 },
    ];
    ```
    Title and SKU are equally important for precise matching, description provides context, category is a fallback.
  </Tab>
  
  <Tab title="Documentation search">
    ```typescript
    const keys = [
      { name: "title", weight: 5 },
      { name: "headings", weight: 3 },
      { name: "content", weight: 1 },
    ];
    ```
    Page titles matter most, then section headings, then body content.
  </Tab>
</Tabs>

## Weight calculation internals

For single-term queries, PicoSearch calculates a weighted average:

```typescript
// Simplified pseudocode from src/pico.ts:49-51
const scores = values.map((v) => fuzzyScore(v, searchTerm));
const weights = normalizedKeys.map((k) => k.weight);
const similarity = weightedAverage(scores, weights);
```

This means a high score in a heavily weighted field can compensate for low scores in other fields.

<Warning>
Extremely high weights (e.g., 100:1 ratio) can cause matches in low-weight fields to be effectively ignored. Use moderate ratios (2:1 to 5:1) for balanced results.
</Warning>
